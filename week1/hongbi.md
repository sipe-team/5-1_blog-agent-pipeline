# Week 1 - hongbi

## 아웃풋 목표

> 이번 파이프라인의 최종 결과물

- 한 주 동안 쌓인 Claude Code 세션 기록, 노션 메모를 자동으로 정리해 테크 블로그 글로 발행하는 시스템
- 최종 발행처: 직접 개발한 블로그 (수동 승인 후 발행)

## 파이프라인 설계

> 전체 흐름

```
[수집] → [요약/정형화] → [초안 저장] → [검토 UI] → [발행]
```

- **수집 소스**: Claude Code 로컬 세션 로그 (`~/.claude/projects/` JSONL), Notion 페이지
- **사용 툴/프레임워크**: Node.js + TypeScript, Anthropic SDK, Supabase, crontab (로컬 자동 실행)
- **발행 채널**: 직접 개발한 Next.js 블로그, 수동 승인 후 포스팅

### 단계별 구현 계획

**1단계 — 수집 (collect.ts)**
- `~/.claude/projects/` 하위 JSONL 파일 탐색
- 이번 주 날짜 범위에 해당하는 메시지만 필터링
- Notion API로 이번 주 수정된 페이지 내용 가져오기

**2단계 — 요약/정형화 (summarize.ts)**
- 수집된 대화 내용을 Claude API에 전달
- 블로그 글 유형(경험/회고 vs 기술 학습)을 자동 판단해 해당 템플릿 형식으로 초안 생성
- 결과를 JSON 배열로 반환 (`title`, `description`, `content`, `tags`)

**3단계 — 초안 저장 (upload-drafts.ts)**
- 생성된 초안을 Supabase `draft_posts` 테이블에 `status: draft`로 저장

**4단계 — 검토 UI (블로그 `/admin/drafts`)**
- 저장된 초안 목록을 카드 형태로 표시
- 제목/태그/내용 직접 수정 가능
- 발제 버튼 클릭 시 블로그 DB에 포스트 생성 후 발행

**자동화**
- 로컬 crontab으로 주 1회 1~3단계 자동 실행
- 실행 후 블로그에서 4단계만 수동으로 진행

### 주요 설계 결정

- **실행 환경**: GitHub Actions 대신 로컬 crontab 선택
  - `~/.claude/` 로그가 로컬에만 존재하기 때문에 서버 실행 불가
  - 맥북이 켜져 있는 동안 주 1회 자동 실행

- **별도 배포 없음**: 기존 블로그에 draft 테이블 + 관리 페이지만 추가
  - 새 프로젝트 배포 비용 없이 기존 인프라 재활용

- **반자동 구조**: 완전 자동 발행이 아닌 AI 초안 생성 → 사람이 검토 후 발행
  - 콘텐츠 품질 유지를 위해 의도적으로 수동 승인 단계 포함

### 블로그 글 유형 템플릿 설계

AI가 일관된 형식으로 글을 생성할 수 있도록 2가지 유형 정의

**유형 1 — 경험/회고 글** (프로젝트, 협업, 이벤트 후기)

```
헤더 없이 도입 문단 (이 글을 쓰게 된 계기)

## 섹션 제목
내용...

회고 또는 앞으로의 계획으로 마무리
```

**유형 2 — 기술 학습 정리 글** (개념 공부, 라이브러리 분석, 트러블슈팅)

```
> 이 글을 쓰게 된 동기/계기 한두 문장

# 개념 소개
내용...

# 마치며
공부하면서 깨달은 점, 앞으로 적용할 점

#### Reference
[참고 링크](URL)
```
