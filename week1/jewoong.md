## 파이프라인 자동화할 Tasks
- 일 단위로 산재되어 있는 업무 내용을 수집/정리
- 수집/정리된 업무내용을 수치/성과화하여 재정리
- 평범한 업무 내용을 성과로 만들 수 있도록 돕는 개선 방향 / 추가로 필요한 정보 피드백 제시

<br/>

### 일 단위로 산재되어 있는 업무 내용을 수집/정리

- 업무를 진행하며 유의미한 정보가 생성/저장되는 Ingestion Source 정리
    - 노션
        - 일일 업무일지
        - 업무 task 단위 진행 내용 정리 페이지
        - 회의록
    - AI Agent
        - CODEX CLI (실제 IDE 상에서 작업 플랜 수립 및 개발 진행)
        - CLAUDE CLI (기타 검색 및 리서치)

- 파이프라인 트리거/스케쥴링
  - 노션은 밤 12시에 전날 생성/수정된 문서들을 기준으로 내용 수집
  - AI Agent들은 밤 12시에 전날 수행된 채팅 내역 텍스트들을 기준으로 내용 수집

- 수집된 정보들은 정제하여 지정된 로컬 경로에 날짜 별 파일로 저장


<br/>


### 수집/정리된 업무내용을 수치/성과화하여 재정리

- 수치 / 성과화 작업을 수행할 프롬프트 템플릿 기반으로 앞 단계에서 정리한 업무 내용들을 가공
  - STAR 기법, XYZ 기법, 수치화 예시 항목들 등을 포함

- 수치/성과화된 결과들을 따로 저장
  - 결과마다 source가 된 업무 내용 정리 파일과 부분을 출처로 명시
  - 미완료 상태이거나 미흡한 성과들은 따로 표기를 하도록 설계
  - ⚠️업무를 진행하면서 작업 내용들이 일 단위로 항상 딱딱 떨어지지 않을텐데 이를 어떻게 같은 항목으로 묶어서 업데이트를 하도록 할지?


<br/>


### 평범한 업무 내용을 성과로 만들 수 있도록 돕는 개선 방향 / 추가로 필요한 정보 피드백 제시

- 앞 단계에서 정리된 성과들을 대상으로 피드백으로 작성해서 다음 날 아침 슬랙으로 전송
  - 미완료/미흡 상태인 성과들을 어떤 방향으로 완료시키면 좋을지
  - 완료 상태이지만 추가적으로 개선하거나 확인해보면 좋을 거 같은 부분들
